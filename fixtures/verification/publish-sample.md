# 一篇接近真实发布的文章

副标题：验证从 Obsidian 到公众号的主链路是否顺畅。

## 开头

如果你经常在 Obsidian 里写作，再手动复制到其他编辑器排版，流程会很碎。  
这篇样例主要用于检查“复制到公众号”的主流程是否已经够顺手。

## 为什么要做本地渲染

过去常见做法是调用远端 API 完成转换，但这会带来三个问题：

1. 需要额外依赖网络。
2. 主题切换通常不是本地即时生效。
3. 图片和 Obsidian 语法兼容很容易失控。

## 一个引用块

> 真正要验证的不是“能不能生成 HTML”，而是“复制后能不能直接发公众号”。

## 一个表格

| 检查项 | 预期 |
| --- | --- |
| 复制状态条 | 明确显示成功、风险或降级 |
| 发布前检查 | 能列出本地图片、缺失附件、不支持语法 |
| 粘贴效果 | 标题、列表、表格、引用基本保持 |

## 结尾

最后请把这篇文章直接复制到公众号写作区域，确认：

- 标题层级是否保留
- 表格是否还在
- 引用块是否可读
- 嵌入内容是否只展开目标部分
