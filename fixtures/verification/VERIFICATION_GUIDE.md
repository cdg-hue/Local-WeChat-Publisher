# 验证说明

建议按下面顺序验证：

## 1. 基础渲染

打开 `basic.md`，点击“复制到公众号”，检查：

- 状态条是否显示“复制成功”
- 预览中的标题、列表、表格、代码块是否正常
- 粘贴到富文本编辑器后是否保留结构

## 2. Obsidian 语法

打开 `obsidian-syntax.md`，点击“复制到公众号”，检查：

- `[[wiki link]]` 是否按文本展示
- `![[embed-note.md]]` 是否整篇展开
- `![[embed-note.md#目标段落]]` 是否只展开指定标题块
- callout 是否降级成引用块
- 本地图片和缺失附件是否出现在“发布前检查”中
- Mermaid / 数学公式是否出现 warning

## 3. 真实发布流

打开 `publish-sample.md`，点击“复制到公众号”，然后：

1. 看状态条显示什么
2. 看发布前检查是否有风险
3. 去微信公众号写作区域直接粘贴
4. 观察标题、表格、引用、嵌入内容是否符合预期

## 4. 反馈给我时建议提供

- 你测试的是哪篇样例
- 状态条显示什么
- 发布前检查里列了哪些问题
- 粘贴到公众号后哪一块不对
- 对应的 Markdown 片段是什么
